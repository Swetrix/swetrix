Swetrix CAPTCHA is a fully open source, cookieless, and privacy-friendly CAPTCHA service. It is designed to be used on websites and web applications to prevent bots from performing automated actions.

## How it works

Unlike traditional CAPTCHAs that require users to identify objects in images or solve puzzles, Swetrix CAPTCHA uses a **Proof of Work (PoW)** mechanism.

<Steps>
  <Step>
    ### Challenge Request When a user interacts with a protected form, the client requests a
    challenge from the Swetrix server.
  </Step>
  <Step>
    ### PoW Calculation The client-side script performs a cryptographic calculation (Proof of Work)
    in the background. This requires a small amount of computational power from the user's device,
    which is trivial for a legitimate user but computationally expensive for bots trying to spam at
    scale.
  </Step>
  <Step>
    ### Verification The solution to the PoW challenge is sent back to the server for verification.
  </Step>
</Steps>

This approach provides a seamless user experience (no more "select all traffic lights") while effectively deterring automated bots.

## Difficulty modes

CAPTCHA projects can use either **Manual** or **Auto** difficulty.

- **Manual** always uses the configured Proof of Work difficulty.
- **Auto** selects the challenge difficulty dynamically from privacy-safe risk signals, such as short-lived project traffic spikes, country-level spikes, replay attempts, validation failures, hosting IPs, suspicious headers, or low pass ratios.

Auto difficulty uses bounded Redis counters with short TTLs and stores only coarse analytics metadata such as the selected difficulty and the reason category that caused a raise. It does not store raw IP addresses in analytics.
